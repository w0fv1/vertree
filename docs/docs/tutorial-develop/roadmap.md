# 后续规划

`0.10.x` 阶段之后，Vertree 的重点不再是“把桌面主流程接上”，而是继续把现有能力打磨得更稳定、更容易验证和分发。

## 1. 版本树交互与性能

- 优化复杂树结构下的布局稳定性
- 继续改进画布拖拽、缩放和连接线绘制
- 降低大树场景下的重绘压力

## 2. 平台分发与集成细化

- Windows 11 新菜单分发体验继续优化
- macOS 签名、notarization 与分发说明补齐
- Linux 非 GNOME 环境的退化体验继续整理
- 各平台权限请求尽量做到按需触发

## 3. 文件能力增强

- 版本搜索
- 标签/备注管理增强
- 差异展示能力探索

## 4. 本机自动化能力

- 扩展本机 HTTP API 的验证场景
- 让开发控制脚本和 API 更适合自动化回归
- 统一桌面 UI、CLI、HTTP API 三条入口的行为约束

## 5. 文档与发布流程

- 继续保持 README、站点文档和 Release Notes 同步
- 让版本号、发布产物和官网首页信息始终一致
- 持续补齐平台差异、依赖和排障说明
